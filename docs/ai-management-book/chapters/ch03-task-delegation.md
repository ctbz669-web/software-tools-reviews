# 第 3 章：AI 任務委派原則——什麼該交給 AI，什麼不該

---

## 先問自己一個問題

在把任務交給 AI 之前，你先問自己這一個問題：

> 「這個任務，最核心的價值是什麼？」

如果答案是「創意」或「判斷」，那 AI 可以幫你加速，但主體要你來。

如果答案是「資訊收集」或「大量處理」或「初稿」，那 AI 可以幾乎完全取代你。

---

## 任務分類框架

### 類型一：資訊彙整型（AI 最擅長）

**定義**：大量閱讀、提取、重組資訊的工作。

**適合交給 AI**：
- 總結一篇十頁的文件
- 翻譯一篇英文文章
- 比較十個競爭商品的規格
- 列出產業前三名的營收數據

**你的角色**：檢視與修正

**警告**：AI 可能會「幻覺」事實，數字類務必二次確認。

### 真實案例

```
任務：比較市面上 10 款 CRM 軟體的功能和價格

交給 AI：
✅ 快速給你一個完整的比較表
✅ 幫你整理每個軟體的優劣勢

你需要做：
⚠️ 二次驗證價格資訊（可能不精確）
⚠️ 根據實際需求選擇，不是只看功能多少
```

---

### 類型二：初稿生成型（AI 擅長，但需要引導）

**定義**：有明確格式或框架要求的文字生成。

**適合交給 AI**：
- 商業 email 初稿
- 會議紀錄格式整理
- 求職信的各版本
- 產品文案的不同風格版本

**你的角色**：提供清晰的格式框架，加上你的個人風格。

### 真實案例

```
任務：寫一封向國外客戶解釋延遲交貨的郵件

❌ 隨便問：
「幫我寫一封郵件」
→ AI 給了一封很正式但冷冰冰的郵件

✅ 給框架：
「我要寫一封郵件給我們最重要的德國客戶，
解釋因為供應商問題導致交貨延遲兩週。

語氣要求：
- 專業但誠懇
- 不要過度道歉（德國客戶重視效率）
- 要提出具體的解決方案

格式要求：
- 不超過 150 字
- 開頭直接說重點
- 結尾提出新的時間表並請對方確認

禁止：
- 不要說「深感遺憾」這類過度道歉的話
- 不要給模糊的時間承諾
」
→ AI 給了一封精準、得體的郵件
```

---

### 類型三：創意發想型（AI 可以加速發散，人類做收斂）

**定義**：需要大量創意、點子的任務。

**適合交給 AI**：
- 頭腦風暴，列出一百個可能的產品名稱
- 發想十種不同的內容切入角度
- 為一個問題找出十種解法

**你的角色**：從 AI 提供的創意中，選擇和組合最適合的。

### 真實案例

```
任務：為一個新的健身 App 想推廣標語

讓 AI 發想 50 個標語方向：
✅「懶人健身法：每天十分鐘，告別健身房」
✅「健身不累：用碎片時間，養成年體態」
✅「你不動，錢包動：懶人理財健身術」

人類收斂：
→ 選擇 3 個最符合品牌調性的
→ 測試哪一個最有記憶點
→ AI 的創意是起點，人類的判斷是終點
```

---

### 類型四：需要判斷的決策型（AI 輔助，人類決定）

**定義**：涉及主觀判斷、價值取捨、風險承擔的決策。

**不適合交給 AI**：
- 「我應該接受哪個 offer？」
- 「這筆投資值不值得？」
- 「這個策略要不要執行？」

**AI 可以做的**：
- 幫你列出「選擇這個的好處和壞處」
- 幫你分析「這個決定可能造成的影響」
- 幫你評估「需要什麼資訊才能做決定」

但最終的判斷，還是你的。

### AI 輔助決策的正確方式

```
❌ 錯誤：
User：「我應該接受哪個工作？」
AI：「我建議選擇工作 A，因為...」
User：好，就工作 A。（直接採用 AI 建議）

✅ 正確：
User：「我正在考慮兩個工作機會：
工作 A：薪資高，但產業成熟，增長有限
工作 B：薪資低，但產業新穎，增長潛力大

請幫我分析這兩個選擇的：
1. 五年後的發展曲線可能長怎樣？
2. 每個選擇的風險點是什麼？
3. 我應該收集什麼資訊來做這個決定？」

AI：提供分析框架和需要考慮的維度

User：（根據 AI 的分析，自己做出判斷）
```

---

### 類型五：複雜人際型（AI 真的不擅長）

**定義**：需要理解人情世故、辦公室政治、上下文理解的任務。

**不適合交給 AI**：
- 處理投訴
- 談判
- 績效面談
- 團隊衝突處理
- 跨部門協調
- 敏感人事決定

**AI 可以做的**：幫你演練可能的情境、準備開場白的選項。

### 為什麼 AI 不擅長這些？

因為這些任務需要：

```
1. 讀空氣——感受現場氣氛，調整說話方式
2. 讀肢體語言——觀察對方的表情、姿態、語氣
3. 讀政治——知道誰說了算、誰是誰的人
4. 即時判斷——當場決定要強硬還是退讓
5. 承擔後果——談判失敗了，你要負責
```

AI 沒有辦法「感受」房間裡的緊張氣氛，也沒有辦法為你做的決定承擔後果。

### AI 幫你演練，但人類上陣

```
任務：下週要跟供應商談判降價

AI 可以幫你準備：
✅ 分析供應商的可能的底線
✅ 準備開場白的三種版本（強硬/溫和/務實）
✅ 預想對方可能提出的反對意見
✅ 準備應對方案和讓步空間
✅ 演練談判破局的替代方案

AI 無法幫你：
❌ 感受對方的肢體語言
❌ 判斷什麼時候該堅持、什麼時候該讓步
❌ 當場承擔談判結果
```

### 人機協作的正確方式

```
Step 1：讓 AI 幫你準備
→「請幫我分析跟這個供應商談判時，可能有三種情境及應對方式」

Step 2：讓 AI 幫你演練
→「請扮演這個供應商，進行一次模擬談判」

Step 3：人類上場
→ 根據 AI 的分析和演練，帶著準備好的方案去談判
→ 現場觀察、即時判斷、當場決策

Step 4：回顧與記錄
→ 回來後讓 AI 幫你記錄這次談判學到的東西
→ 下次談判時，這些經驗可以成為新的 Context
```

**記住：AI 是你的戰前教練，不是你的談判替代者。**

## 常見錯誤：把判斷型任務交給 AI

### 錯誤一：讓 AI 幫你選投資標的

```
❌「哪支股票值得買？」
→ AI 給了你一個看起來很專業的分析，
  但它不知道你的風險承受能力、
  你的投資目標、市場的非理性波動

✅「分析這三支股票的風險和機會，
  我應該收集什麼資訊來做判斷？」
→ AI 幫你建立評估框架，你來做決定
```

### 錯誤二：讓 AI 幫你決定策略

```
❌「我們公司應該進入哪個市場？」
→ AI 給了你一個漂亮的策略報告，
  但它不知道你公司的組織能力、
  現金儲備、政治環境

✅「影響這個市場進入決策的關鍵維度有哪些？」
→ AI 幫你盤點維度，你來整合所有資訊做判斷
```

## 任務委派的常見誤區

### 誤區一：「AI 做得比我好」

有時候是對的，有時候不是。

**解決方法：先用 BCG 矩陣判斷任務類型，再决定要不要給 AI。**

### 誤區二：「全部自己做比較快」

有些任務 AI 五分鐘可以完成，你卻花了一小時。

**解決方法：記錄你每週手動做的重複性任務，評估哪個可以交給 AI。**

### 誤區三：「一次問清楚」

有些人希望一次給 AI 所有背景，然後得到完美答案。

**解決方法：複雜任務分階段給，AI 表現會更好。**

## 任務委派的進階技巧

### 技巧一：任務拆分

不要一次給一個大任務，拆成幾個小任務。

```
❌ 一次給：
「幫我從零開始做一個市場推廣計劃」

✅ 拆成幾次：
Step 1：「幫我想想這個產品的三個主要賣點」
Step 2：「基於這三個賣點，幫我設計一個內容策略」
Step 3：「根據內容策略，幫我列出一個月的內容行事曆」
Step 4：「針對第一週的內容，幫我寫三篇社群貼文」
```

### 技巧二：給 AI 角色

賦予 AI 一個專業身份，輸出品質會大幅提升。

```
❌ 沒有角色：
「幫我分析這篇文章」

✅ 有角色：
「你是一個有 10 年經驗的內容行銷總監。
 請從這個角度分析這篇文章的優缺點，
 並提出可以改進的具體建議。」
```

### 技巧三：要求 AI 提問

讓 AI 在回答之前，先問你一些關鍵問題。

```
 Prompt：
「在我給你這個任務之前，請先問我三個問題，
 確保你完全理解我的需求後再開始回答。」

好處：
- AI 會問出你沒有想到的必要背景
- 避免做到一半才發現方向錯誤
- 節省來回修正的時間
```

## 練習：盤點你的工作任務

拿出你的月曆或待辦清單，問自己每一項：

1. 這個任務，核心價值是什麼？
2. 有大量資訊需要處理嗎？
3. 有明確的格式或框架嗎？
4. 需要我的主觀判斷嗎？
5. 需要理解人情世故嗎？

```

任務三：[     ] → 類型：___ → AI 適合度：___
```

## 本章重點

1. 任務委派前先問自己：「核心價值是什麼？」
2. 五種任務類型：資訊彙整、初稿生成、創意發想、決策建議、複雜人際
3. AI 最強的是「大量資訊處理」和「有框架的文字生成」
4. 人類的核心價值在「判斷」和「創意收斂」
5. 常見錯誤：把判斷型任務交給 AI、AI 輔助決策的正確方式
6. 任務委派三個誤區：過度依賴 AI / 全部自己做 / 一次問清楚
7. 進階技巧：任務拆分 / 給 AI 角色 / 要求 AI 提問
