# 第 4 章：AI OKR——如何給 AI 下精準指令

---

## 多數人給 AI 的指令，其實是無效指令

讓我做一個實驗。

請現在心裡想一個問題，然後想想你會怎麼問 ChatGPT。

大多數人的問法，會是這樣的：

> 「幫我寫一篇關於ESG的文章」

這句話聽起來很具體，但其實非常模糊。

什麼是「ESG」？是給投資人看的？給員工看的？給大眾看的？要長還是要短？要專業論文型還是大眾科普型？

**你以為你說了「一篇關於 ESG 的文章」，AI 就知道你想要什麼。**

**但 AI 其實不知道。**

---

## 一個好的 AI 指令 = OKR

讓我把 OKR 的概念，直接套用在給 AI 下指令：

### O（Objective）= 你的目標

> 「我要這篇文章用來做什麼？」

**不是**「寫一篇 ESG 文章」
**而是**「這週要在公司內部分享，作為推動 ESG 的內部溝通素材」

### KR（Key Results）= 可量化的交付標準

| 維度 | 問題 |
|------|------|
| **讀者是誰** | 內部員工？投資人？一般大眾？ |
| **長度** | 500 字以內？還是 2000 字？ |
| **風格** | 專業嚴謹？輕鬆易懂？感性訴求？ |
| **核心訊息** | 最希望讀者記住的一件事是什麼？ |
| **格式** | 純文字？有標題？要分段落？要 bullet points？ |
| **禁止** | 特別不想要什麼？（例如：不要太多術語） |

## 四種常見指令類型的正確模板

### 類型一：商業報告

```
【情境】
我是某科技公司的公關經理，需要向董事們匯報公司 ESG 進展。

【目標】
寫一份半頁的摘要，讓董事們快速了解公司 ESG 的三大重點。

【要求】
- 三大重點，每點不超過三句話
- 用數據說話，附上關鍵數字
- 結尾呼籲董事支持即將推動的綠色能源計劃

【禁止】
不要超過半頁。避免過多專業術語。
```

#### 實際案例

```
❌ 錯誤示範：
「幫我寫一個 ESG 報告」

✅ 正確示範：
「我需要向公司董事們匯報 ESG 進展，
目標是讓他們支持即將推動的綠色能源計劃。

要求：
- 半頁以內（他們時間有限）
- 三大重點，每點附數據
- 語氣專業但有說服力
- 結尾要有明確的 Call to Action

禁止：
- 不要寫成學術論文
- 不要超過半頁
- 不要出現我們還沒有數據支撐的承諾
」
```

### 類型二：創意發想

```
【情境】
我需要為一個新的健身 App 想宣傳標語。

【目標】
頭腦風暴 20 個不同角度的標語方向。

【要求】
- 前 10 個：強調效果（如：減肥、變有力）
- 後 10 個：強調感受（如：自由、快樂、釋壓）
- 每個標語不超過 10 個字

【目標讀者】
25-35 歲的上班族，時間不多但想要改變。
```

#### 實際案例

```
❌ 錯誤示範：
「幫我想一些健身 App 的標語」

✅ 正確示範：
「我需要為一個健身 App 想標語。

目標讀者：25-35 歲台北上班族，時間碎片化，想要改變體態但不想去健身房

標語方向：
1. 強調「懶人也能健身」（10 個）
2. 強調「省時」（10 個）
3. 強調「在家就能做」（10 個）

每個標語不超過 10 個字。
要有記憶點，讓人想點進去了解。
」
```

### 類型三：郵件撰寫

```
【情境】
我要寫一封郵件給國外的合作窗口，討論延後專案交付時間。

【目標】
有禮貌但堅定地表達延後的需求，並提出新的時間建議。

【要求】
- 總長度不超過 150 個字
- 開頭自我介紹並說明原因（中午前提供給你們）
- 結尾提出新的交付時間，並請對方確認

【語氣】
專業、誠懇、不卑不亢。讓對方感受到我們的重視，但也不需要過度道歉。
```

#### 實際案例

```
❌ 錯誤示範：
「幫我寫一封延後專案的郵件」

✅ 正確示範：
「我要寫一封郵件給德國的合作方，告訴他們專案要延後兩週。

背景：
- 我們的供應商出了問題
- 這是一個長期合作關係，我們不希望破壞信任
- 德國人重視效率和明確的時間承諾

要求：
- 150 字以內（他們不喜歡看長郵件）
- 開頭直接說重點
- 明確提出新的時間表
- 語氣：專業、誠實、不卑微

禁止：
- 不要過度道歉（德國人不 care）
- 不要只說「有問題」，要說「出了什麼問題」
- 不要沒有新的時間承諾
」
```

### 類型四：翻譯潤稿

```
【情境】
我有一段中文摘要，需要翻成英文給國外期刊使用。

【原文】
[在這裡貼上中文摘要]

【目標】
翻譯成學術英語，適合給國外審稿者閱讀。

【要求】
- 保持學術語氣，不要太口語化
- 專有名詞第一次出現時括號標注原文
- 長度與原文相近，不要過度延伸

【特别要求】
我需要這個翻譯在學術搜尋中容易被找到，請選擇用詞精準的翻譯。
```

## 不要問「能不能」，直接假設「可以」

很多人在問 AI 的時候，會先問：

> 「能不能幫我翻譯這段？」
> 「你能不能幫我寫這個？」

浪費了第一句的機會。

AI 不會說「不能」。它只會試圖回答。

所以與其問「能不能」，不如直接給指令。

---

## OKR 指令檢查清單

給 AI 下指令前，快速檢查：

```
□ O（目標）：我明確了這個任務的用途
□ KR1：讀者確定了
□ KR2：長度確定了
□ KR3：風格確定了
□ KR4：核心訊息確定了
□ KR5：格式要求確定了（沒有的話也標注出來）
□ KR6：禁止事項標注了（有的話）
```

## 常見的 Prompt 錯誤與修正

### 錯誤一：背景太少

```
❌「幫我寫一個產品介紹」
→ AI 給了一個很通用的產品介紹

✅「幫我寫一個產品介紹，這個產品是：
- 針對 25-35 歲的台北女性
- 主要解決她們時間碎片化的健身需求
- 競爭對手是 XXX，我們的差異化是 YYY
- 行銷檔次：中高價位
」
→ AI 給了一個更精準的產品介紹
```

### 錯誤二：沒有指定輸出格式

```
❌「分析一下這個產業」
→ AI 給了一堆文字，你還要自己整理

✅「用表格分析，欄位包括：市場規模、主要競爭者、市場成長率、
進入壁壘。每個欄位不超過 30 個字。」
→ AI 給了一個你可以直接用的表格
```

### 錯誤三：忘記設禁止事項

```
❌「幫我寫一篇商業 email」
→ AI 給了一封很完美的 email，但你發現它太正式了，
  不適合你們公司的文化

✅「幫我寫一篇商業 email
風格：輕鬆但專業（我們公司文化不太正式）
長度：100 字以內
禁止：
- 不要用「以此類推」、「本著」這類正式商業用語
- 不要有過度商務的罐頭結尾
」
→ AI 給了一封風格正確的 email
```

## 逆向 Prompt：先想輸出，再想輸入

有時候你不知道怎麼下指令，因為你不確定需要什麼。

這個時候，用「逆向 Prompt」的方法：

### 步驟

```
Step 1：先想清楚你要的輸出是什麼
→ 我想要一個 PPT 大綱，包含 5 頁

Step 2：定義每頁的內容
→ 第一頁：標題和副標題
  第二頁：問題和背景
  第三頁：解決方案
  ...

Step 3：根據輸出，反推需要什麼輸入
→ 我需要提供這個 PPT 的目標讀者和核心訊息
```

### 逆向 Prompt 示例

```
❌ 正向思考（不知道怎麼問）：
「幫我做一個簡報」

✅ 逆向思考：

輸出目標：
→ 5 分鐘的投資人簡報，10 頁以內

每頁內容：
→ 第 1 頁：標題頁
→ 第 2 頁：我們在解決什麼問題
→ 第 3 頁：市場有多大
→ ...

輸入確認：
→ 我需要告訴 AI：目標讀者是誰、核心訊息是什麼、有什麼限制

完整 Prompt：
「我要做一個 5 分鐘的投資人簡報，目標讀者是天使投資人，
他們關心：市場規模、商業模式、團隊能力。
核心訊息：我們是第一個針對「懶人健身」的解決方案。
請給我 8 頁的大綱，每頁用一句話說明。」
```

## 給 AI 角色：提升輸出品質的技巧

給 AI 一個專業身份，往往能大幅提升輸出品質。

### 簡單的角色賦予

```
❌ 沒有角色：
「幫我分析這篇文章」

✅ 有角色：
「你是一個資深的內容編輯，請分析這篇文章的：
1. 標題的吸引力
2. 結構的邏輯性
3. 語氣的一致性
4. 改進的具體建議」
```

### 複雜的角色賦予

```
你是一個有 15 年經驗的國際品牌策略顧問。
你有以下背景：
- 曾在麥肯錫工作 8 年，專精消費品品牌
- 幫助過 30 多個品牌從零開始建立市場定位
- 專精「懶人經濟」和「小確幸」市場

請用這個身份，分析我們品牌的定位策略。
```

## 處理 AI 的不完全回答

有時候 AI 給的答案不完整，這個時候要知道怎麼修正。

### 情況一：AI 停下來了

```
Prompt：「繼續」
或
Prompt：「請繼續完成上一個回答」
```

### 情況二：AI 給了一部分

```
Prompt：「很好，但我只需要前三個。請基於這些，給我一個詳細的執行計劃」
```

### 情況三：AI 給的偏離主題

```
Prompt：「你剛才的回答偏離了主題。我要的是[具體說明]。請重新回答」
```

## 本章重點

1. **模糊指令 = 模糊結果。給 AI 的指令要像 OKR 一樣清晰。**
2. **一個好的指令需要：情境（Context）+ 目標（Objective）+ 要求（Requirements）+ 禁止（Don'ts）**
3. **四種常見指令模板：商業報告、創意發想、郵件撰寫、翻譯潤稿**
4. **不要問「能不能」，直接給指令**
5. **OKR 指令檢查清單：6 個維度快速檢查**
6. **逆向 Prompt**：先想輸出格式，再反推需要什麼輸入
7. **給 AI 角色**：提升輸出品質的技巧
