# 第 7 章：AI Prompt 工作流設計——把流程 SOP 化

---

## 為什麼你的 AI 總是要重新開始？

你有沒有過這樣的經驗？

每次要用 AI 處理同一類型的任務時，你都要重新想一遍「上次是怎麼問的」。有時候想起來，有時候想不起來。

於是每次的結果，品質參差不齊。

這是因為你沒有把流程 SOP 化。

---

## 把 AI 工作流 SOP 化的三個層次

### 層次一：Prompt 模板

把常見任務的「標準指令」寫成填空式模板，下次只需要填空。

```
【Email 潤稿模板——直接複製使用】

---
【情境】
我是______（你的職位），要寫一封______（類型）的郵件。

【目標受眾】
______（誰會讀這封信？）

【郵件目的】
______（主要訊息）

【語氣】
______（正式/輕鬆/親切/強硬）

【禁止事項】
______（特別不想出現的內容）

---

【原稿】
______（在這裡貼上原文）
---
```

#### Prompt 模板的實際案例

```
案例：商業 email 潤稿

模板 v1.0：
【情境】我是產品經理，要寫一封郵件給國外客戶
【目標】討論專案延後的事項
【語氣】專業但誠懇
【禁止】過度道歉、不確定的承諾

→ AI 潤稿後的版本，總是太正式或太隨意

模板 v2.0（改善版）：
【情境】我是產品經理，要寫一封郵件給德國客戶
【目標】討論專案延後的事項（預計延後兩週）
【語氣】德國人重視效率和明確的時間承諾，不要過度道歉
【禁止】不要說「深感遺憾」，不要模糊承諾
【原稿】
[貼上原文]

→ AI 潤稿後的版本，精準符合需求
```

---

### 層次二：多步驟工作流

有些任務不能一步到位，需要拆成幾個階段：

```
【文章寫作工作流】

Step 1：大綱生成
 Prompt：「根據以下主題，生成一個五段式的大綱..."
  輸出：大綱草稿

Step 2：大綱確認
 你：審視大綱，修改確認

Step 3：各段撰寫
 Prompt：「根據以下大綱，撰寫第一段的內容..."
  對每個段落重複

Step 4：統合潤稿
 Prompt：「以下是全文，請幫我統一風格、修正矛盾..."
```

#### 多步驟工作流的實際案例

```
案例：市場分析報告

Step 1：確定分析框架
 Prompt：「我要分析 [產業] 的市場概況。
 請建議一個包含 5 個維度的分析框架。」
 AI 輸出：市場規模、競爭格局、發展趨勢、
         進入壁壘、風險因素

Step 2：對每個維度進行分析
 Prompt：「請分析 [產業] 的市場規模，
 包含：近三年數據、主要驅動因素、預測。」
 對每個維度重複

Step 3：整合成報告
 Prompt：「以下是各維度的分析結果，
 請整合成一份完整的市場分析報告，
 包含執行摘要和結論建議。」
```

---

### 層次三：反饋循環

建立一個「AI 回答 → 你審核 → 修正指令 → 再問」的循環：

```
第 1 次：AI 回答
 ↓ 不滿意？
第 2 次：提供更具體的修正指令
 ↓ 不滿意？
第 3 次：再修正
 ↓ 直到滿意
```

記錄每次的「修正指令」，未來可以逐漸優化成模板。

#### 修正指令的範例

```
第 1 次：Prompt：「幫我寫一個產品介紹」
 AI 輸出：太過技術導向

第 2 次：Prompt：「太過技術導向，請用白話文，
 目標讀者是 30-40 歲的普通消費者」
 AI 輸出：好一點，但太長

第 3 次：Prompt：「很好，但請精簡到 200 字以內，
 並在第一句話就說明這個產品解決什麼問題」
 AI 輸出：滿意的版本

記錄下來：
「產品介紹 Prompt = 產品描述 + 白話要求 + 
 目標讀者 + 字數限制 + 開頭要求」
```

## 每個好結果，都問一句：「這個能當模板嗎？」

這是 Prompt 迭代最重要的習慣。

當 AI 給了你一個滿意的答案，不要只是複製貼上就走人。問它一句話：

```
「這個任務的 Prompt 是什麼？未來我怎麼才能重現這個結果？」
```

AI 會幫你分析這個好結果背後的 Prompt 邏輯，生成一個可複用的模板。

---

### 這個模式為什麼有效？

```
你：幫我寫一封郵件
AI：[完成得很好]

你：「這個郵件的 Prompt 是什麼？未來我怎麼才能重現這個結果？」

AI：「這是這個郵件的 Prompt 模板：
【情境】我是[職位]，要寫一封[類型]郵件
【目標】[主要訊息]
【風格】[專業/輕鬆/...]
【禁止】[特別不想出現的內容]
」

你：把這個模板記下來，未來直接填空
```

### 讓 AI 幫你建立 Prompt 庫

每次得到好結果時，用這個問題建立你的私人 Prompt 庫：

```
Q：「這個輸出背後的 Prompt 是什麼？可以當成未來類似任務的模板嗎？」
```

一年下來，你會有一個「為你量身打造的 Prompt 模板庫」，而不是每次從零開始。

## 與 OKR 框架的整合

如果你已經讀過第 4 章，你會發現：

> **Prompt 工作流的底層邏輯，其實就是 OKR。**

在第 4 章我們學到：
- **O（Objective）** = 你的目標 = Prompt 中的「情境 + 目標」
- **KR（Key Results）** = 可量化的交付標準 = Prompt 中的「要求 + 禁止」

所以這一章的重點不是學新的框架，而是：**把框架用起來。**

以下是兩個實際應用場景的工作流設計。

---

## 兩個關鍵原則

### 原則一：逆向 Prompt

先想「我最後想要什麼形式的輸出」，再反推「我需要給 AI 什麼輸入」。

```
步驟 1：定義輸出格式（圖？表格？段落？）
步驟 2：定義讀者（誰要看這個？）
步驟 3：定義目標（看完後要採取什麼行動？）
步驟 4：根據以上，反推 Prompt 內容
```

#### 逆向 Prompt 實際案例

```
任務：向老闆匯報本季工作成果

❌ 正向思考（不知道怎麼問）：
「幫我整理一下這季的工作」

✅ 逆向思考：

Step 1：定義輸出形式
→ 一頁 PDF，圖文並茂
→ 包含：關鍵成果（用數字說話）、下季目標、主要挑戰

Step 2：定義讀者
→ 老闆，時間有限，只看重點

Step 3：定義目標
→ 讓老闆認可這季的成果，並支持下季資源

Step 4：反推 Prompt
「我要向老闆匯報這季工作成果。

目標：
- 篇幅：一頁以內（老闆時間有限）
- 成果：用數字說話（營收/用戶/專案數）
- 下季：提出一個需要資源的項目

格式：
- 標題：Q1 工作成果
- 第一部分：三個關鍵成果（每個不超過兩句話）
- 第二部分：一個下季重點項目
- 第三部分：需要什麼資源

禁止：
- 不要流水帳
- 不要只敘述過程，要說成果
」
```

### 原則二：分段交付

不要一次給太多任務。複雜任務分批給，AI 表現會更好。

```
❌ 一次給：「幫我分析這個產業、競爭者、機會、風險，並寫成一份報告」

✅ 分次給：
  Step 1：「先幫我用 SWOT 分析這個產業」
  Step 2：「基於 SWOT，幫我列出三個策略選項」
  Step 3：「幫我把這三個策略選項寫成一份簡潔的報告」
```

## 常見的 Prompt 錯誤

### 錯誤一：貪心，一次給太多

```
❌ 錯誤：
「我要開一個母嬰用品電商網站，幫我想名字、logo、
市場分析、定價策略、行銷計劃...」

→ AI 輸出：每個都很淺

✅ 正確：
分次問：
Step 1：「幫我想 10 個母嬰用品電商網站的名字」
Step 2：「針對第一個名字，幫我做市場分析」
Step 3：「根據市場分析，幫我訂定定價策略」
...如此類推
```

### 錯誤二：忘記設定角色

```
❌ 錯誤：
「幫我寫一篇關於減肥的文章」

→ AI 輸出：通用的健康減肥建議

✅ 正確：
「你是一個有 10 年經驗的健身教練，專精懶人減肥法。
幫我寫一篇關於減肥的文章，
目標讀者是沒有時間運動的上班族。」

→ AI 輸出：針對性的實用建議
```

### 錯誤三：沒有說「不要做什麼」

```
❌ 錯誤：
「幫我寫一篇產品介紹」

→ AI 輸出：可能出現大量專業術語，
  或者太過強調功能而忽略了好處

✅ 正確：
「幫我寫一篇產品介紹
禁止：
- 不要用「奈米技術」、「量子級別」這類空洞術語
- 不要只列功能，要說每個功能帶來什麼好處
- 不要超過 300 字
」
```

## 本章重點

1. **三個層次**：Prompt 模板 → 多步驟工作流 → 反饋循環
2. **OKR 框架**是 Prompt 的底層邏輯（詳細說明見第 4 章）
3. **分段交付**：複雜任務分批給，品質更好
4. **每個好結果都問**：「這個能當模板嗎？」——建立你的私人 Prompt 庫
5. **三個常見錯誤**：貪心一次給太多 / 忘記設定角色 / 忘記說不要做什麼

---

## 💡 反思問題

1. 你有沒有一個任務是「每次都要重複問類似的事情」的？建立模板會有幫助嗎？
2. 你有沒有「試了兩三次都放棄了」的經驗？問題可能不在 AI，在你沒有建立修正的反饋循環。
3. 你有沒有「收到好結果但沒有記下來」的經驗？下次記得問「這個能當模板嗎？」

## 附錄：常用 Prompt 模板庫

以下是可以直接複製使用的模板：

### 模板一：商業郵件

```
【情境】
我是______（職位），要寫一封給______（對象）的郵件。

【目標】
______（這封郵件的主要目的）

【語氣】
______（正式/專業/友善/簡潔）

【禁止】
______（特別不想出現的內容）

【原稿】
______（有的話粘上原文）
```

### 模板二：市場分析

```
【目標】
分析______（產業/市場）是否值得進入

【需要的維度】
1. 市場規模和成長率
2. 主要競爭者和市佔率
3. 進入門檻
4. 風險和機會
5. 建議策略

【公司背景】
______（我們公司的優勢和劣勢）

【禁止】
不要編造數字，所有數字要標注來源
```

### 模板三：創意發想

```
【任務】
幫我想______個______（類型）的點子

【目標受眾】
______（誰會看到/使用這些點子）

【方向要求】
______（重點強調什麼）

【禁止】
______（特別不想出現的方向）
```

---

**使用說明**：把這些模板複製到你的筆記工具（如 Notion、Obsidian），每次用的時候拿出來填空就好。
