## 為什麼要先懂框架？

在開始用管理學來「管」AI 之前，你需要先知道有哪些武器可以用。

**先問自己一個問題：**

你現在做一件工作——假設是「寫一封商業郵件」——要多久？

30 分鐘？一小時？

用對 AI + 對的方法，變成 5 分鐘。

**這就是管理框架的價值：不是讓你學更多工具，是讓你用對方法，把 30 分鐘的工作變成 5 分鐘。**

## 框架一：OKR——目標與關鍵成果

### 傳統管理怎麼說

OKR（Objectives and Key Results）是 Google 推廣的目標管理法。

- **O（Objective）**：你想要達成的大方向
- **KR（Key Results）**：可量化、具體的關鍵成果

### 在 AI 世界裡

當你對 AI 下指令時，其實就在做 OKR：

```
O（目標）：我需要一份市場分析報告
KR（關鍵成果）：
  - 涵蓋近三年趨勢
  - 包含主要競爭者分析
  - 用決策者的視角撰寫
```

問題來了：很多人只給了「O」，沒給「KR」。

所以 AI 只能給你「一份市場分析報告」而不是「你要的那種報告」。

### 常見錯誤 vs 正確做法

```
❌ 錯誤示範：
「幫我分析這個產業」

✅ 正確示範：
O（目標）：評估母嬰用品電商市場是否值得進入
KR1：近三年市場規模成長率
KR2：前三大競爭者的市佔率和營收
KR3：進入門檻和風險
KR4：建議的切入策略
```

## 框架二：MECE——不重複、不遺漏

### 傳統管理怎麼說

MECE（Mutually Exclusive, Collectively Exhaustive）是麥肯錫顧問公司發明的思考框架。

簡單說：**分類的時候，不重複、不遺漏。**

### 在 AI 世界裡

當你要 AI 幫你「分析一個問題」，你可以用 MECE 來下指令：

```
❌ 沒有結構的問法：
「分析一下這個產業的機會和威脅」

✅ 有 MECE 結構的問法：
「用 SWOT 分析，分別說明：
  - S (Strengths)：這個公司的優勢
  - W (Weaknesses)：劣勢
  - O (Opportunities)：外部機會
  - T (Threats)：外部威脅」
```

AI 不知道你要什麼結構，除非你告訴它。

### 常見的 MECE 應用

| 場景 | 問題 | MECE 結構 |
|------|------|-----------|
| 市場分析 | 這市場有什麼機會？ | 內/外部 × 有利/不利 |
| 問題分析 | 為什麼業績下滑？ | 人/事/時/地 |
| 決策評估 | 應該接受哪個 offer？ | 優/劣/風/機 |
| 競爭分析 | 對手的強弱在哪？ | 產品/通路/價格/服務 |

## 框架三：BCG 矩陣——優先順序

### 傳統管理怎麼說

BCG 矩陣用「市場成長率」和「市佔率」把產品分成四類：

- **明星**：高成長、高市佔 → 大力投資
- **金牛**：低成長、高市佔 → 收割現金
- **問題**：高成長、低市佔 → 評估是否值得投入
- **瘦狗**：低成長、低市佔 → 考慮退出

### 在 AI 世界裡

用來決定**把什麼任務交給 AI**：

| 類型 | AI 適合度 | 原因 |
|------|-----------|------|
| 大量資料處理 | ⭐⭐⭐⭐⭐ | AI 最擅長 |
| 需要判斷的決策 | ⭐⭐ | AI 缺乏真實世界經驗 |
| 創意發想 | ⭐⭐⭐⭐ | AI 有大量素材 |
| 複雜人際談判 | ⭐ | AI 不懂人情世故 |

### BCG 在 AI 任務分配中的應用

```
把工作任務放進 BCG 矩陣：

                    高重複性
                         │
         ┌───────────────┼───────────────┐
         │    金牛        │    明星       │
         │  (AI 做)       │  (AI 加速)    │
         │  • 翻譯        │  • 報告生成   │
         │  • 格式整理    │  • 數據分析   │
低判斷難度               ├───────────────┼───────────────┤
         │    瘦狗        │    問題       │
         │  (不要用 AI)   │  (人類做)     │
         │  • 裁員決定    │  • 策略制定   │
         │  • 複雜談判    │  • 複雜判斷   │
         └───────────────┴───────────────┘
                         │
                    高判斷難度
```

## 框架四：SWOT——優勢與劣勢

### 傳統管理怎麼說

SWOT 分析是最常見的策略框架：
- **S (Strengths)**：內部優勢
- **W (Weaknesses)**：內部劣勢
- **O (Opportunities)**：外部機會
- **T (Threats)**：外部威脅

### 在 AI 世界裡

SWOT 是讓 AI 做分析的經典框架，比 MECE 更細緻：

```
請用 SWOT 分析框架分析這家公司：
- S：這家公司的獨特優勢是什麼？
- W：劣勢是什麼？
- O：外部市場有什麼機會？
- T：有哪些威脅和風險？
```

### SWOT 與其他框架的組合

```
讓 AI 用「SWOT + 策略建議」一次完成：

請用 SWOT 分析[公司]，並針對每個維度提出對應策略：

S（優勢）：如何最大化利用？
  → 策略：...

W（劣勢）：如何極小化或改進？
  → 策略：...

O（機會）：如何抓住這些機會？
  → 策略：...

T（威脅）：如何避開或降低風險？
  → 策略：...
```

## 框架五：8020 法則——抓重點

### 傳統管理怎麼說

80% 的成果來自 20% 的努力。找出關鍵少數，忽略無效多數。

### 在 AI 世界裡

用來判斷**什麼值得用 AI 處理**：

```
這個任務的「20%」是什麼？
- 這項工作中，最有價值的部分是什麼？
- 哪 20% 的輸入能產生 80% 的價值？

當你找到這個重點，優先用 AI 處理這 20%。
```

### 8020 在 AI 的實際應用

```
任務：準備一場投資人簡報（預計 20 頁）

用 8020 分析：
- 20% 核心內容：你的獨特價值主張，商務模式、數據（這 20% 決定成敗）
- 80% 輔助內容：團隊介紹、公司歷程、詳細財務（錦上添花）

用 AI 分配：
- AI 處理 80% 輔助內容（格式整理、初稿）
- 人類專注 20% 核心內容（打瘸、論點、數據）
```

## 框架六：STAR——結構化表達

### 傳統管理怎麼說

STAR 是面試和報告的經典框架：
- **S (Situation)**：情境
- **T (Task)**：任務
- **A (Action)**：行動
- **R (Result)**：結果

### 在 AI 世界裡

當你要 AI 幫你結構化表達：

```
用 STAR 框架，回顧你上季的工作：

S（情境）：我在什麼背景下工作？
T（任務）：我被交付什麼目標？
A（行動）：我做了什麼來達成目標？
R（結果）：最終成果如何？有什麼數據證明？
```

### STAR 的應用場景

| 場景 | Prompt |
|------|--------|
| 年度回顧 | 用 STAR 框架寫一篇 500 字的個人年度回顧 |
| 求職自我介紹 | 用 STAR 框架介紹我自己，重點在成就 |
| 專案簡報 | 用 STAR 框架呈現這個專案的价值 |
| 績效面談 | 用 STAR 框架預演一次績效談話 |

## 六個框架的組合應用

多數時候，一個任務需要多個框架組合使用。

### 案例：分析是否進入新市場

```
Step 1：用 BCG 矩陣判斷
→ 這個市場是高成長、高潛力嗎？（機會象限）

Step 2：用 SWOT 分析
→ 我們公司的優劣勢是否支援進入？

Step 3：用 MECE 結構
→ 評估維度：市場/競爭/風險/收益

Step 4：用 8020 法則
→ 最關鍵的 20% 評估維度是什麼？

Step 5：用 OKR 下指令給 AI
→ O：評估進入東南亞市場的可行性
→ KR1：市場規模和成長率
→ KR2：主要競爭者和進入壁壘
→ KR3：預估投資報酬率
→ KR4：風險矩陣和緩解方案

AI 輸出：一份完整的市場進入評估報告
```

## 常見問題：框架要一次用很多嗎？

不需要。

```
原則：用你需要的，不是用全部。

簡單任務：
→ 用 1 個框架（通常是 OKR 或 MECE）

複雜任務：
→ 用 2-3 個框架組合

專案級別：
→ 用全部框架系統性分析
```

---

## 本章重點

| 框架 | 在 AI 的應用 |
|------|-------------|
| **OKR** | 下指令時同時給目標(O)和關鍵成果(KR)，避免模糊指令 |
| **MECE** | 用框架下指令，讓 AI 的回答有結構，避免遺漏 |
| **BCG 矩陣** | 用來判斷什麼任務適合交給 AI，什麼不適合 |
| **SWOT** | 分析框架，讓 AI 做策略分析，可與 OKR 組合 |
| **8020 法則** | 找出最值得用 AI 處理的 20%，優先順序 |
| **STAR** | 結構化表達框架，適用於回顧、簡報、求職 |
