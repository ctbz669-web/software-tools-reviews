# 第 8 章：知識管理與記憶策略——讓 AI 記得該記的

---

## AI 記憶系統來了，但代價是什麼？

這兩年，越來越多的 AI 工具開始有了「記憶系統」。

這看起來很美好——但有一個問題：

> **記憶不是免費的。**

每一次 AI 讀取記憶，都消耗 Token。記憶越多，消耗越快。

---

## 不是所有東西都塞進上下文

### 原因一：Token 不是免費的

當前 AI 的記憶機制，多數是透過「上下文」（Context）實現。你讓 AI 記得越多，每次問問題的成本就越高、速度也越慢。

### 原因二：記憶越多，干擾越多

讓 AI 記住太多不相關的事情，反而會讓它分心，給出不精確的答案。

### 原因三：AI 會遺忘

上下文窗口是有限的。當記憶太多，AI 反而會忘記重要的事情。

---

## 企業管理的答案：分層管理

在企業裡，知識管理從來不是「把所有事情都記住」。

而是分層管理：

```
企業 SOP 分層架構

┌─────────────────────────────────────┐
│          每日工作規範                │  ← 所有員工都會讀
├─────────────────────────────────────┤
│          部門專項 SOP                │  ← 該部門才會讀
├─────────────────────────────────────┤
│          專案執行手冊                │  ← 該專案才會讀
└─────────────────────────────────────┘
```

**AI 的記憶管理，完全一樣。**

---

## AI 記憶分層

```
AI 記憶分層

┌─────────────────────────────────────┐
│      每日上下文（系統層）            │  ← 每個對話都會讀
│  • 你的產業和職業
│  • 常用術語和偏好
│  • 基本的風格指導
├─────────────────────────────────────┤
│      任務模板庫（任務層）             │  ← 進行該類任務時讀取
│  • 這個任務的標準 Prompt
│  • 這個任務的交付格式要求
├─────────────────────────────────────┤
│      專案 Context（專案層）          │  ← 處理該專案時讀取
│  • 這個專案的特殊背景
│  • 這個專案的目標和約束
└─────────────────────────────────────┘
```

---

## 第一層：每日上下文（系統層）

這是每次 AI 對話都會讀取的基礎背景。

### AI 身份卡

建立一份「AI 身份卡」，每個新對話一開始就貼給 AI：

```
【我的基本資訊】
- 產業：_____________
- 職業：_____________
- 負責產品：_____________

【我的溝通風格】
- 報告時偏好：_____________
- 決策時需要的資訊：_____________

【我的目標】
- 這季最重要的目標：_____________
- 老闆最關心的指標：_____________
```

### AI 身份卡的實際案例

```
【Kevin 的 AI 身份卡】

- 產業：母嬰用品電商
- 職業：產品經理
- 主要產品：線上育兒課程平台

- 報告風格：數字導向，一頁以內
- 老闆關心：用戶成長、營收、轉化率

- 這季目標：拓展東南亞市場
- 老闆最關心：Q2 用戶能不能成長 30%
```

## 第二層：任務模板庫（任務層）

每個任務類型建立一個模板，下次只需要填空。

### 任務模板庫的資料夾結構

```
/AI工作模板/
  /01-郵件寫作/
      prompt-template.md
      format-guide.md
      examples/
  /02-市場分析/
      prompt-template.md
      framework-guide.md
      examples/
  /03-簡報設計/
      prompt-template.md
      structure-guide.md
      examples/
```

### 任務模板的範例

```
【任務模板：市場分析】

檔案：prompt-template.md

---
【任務目標】
市場分析報告

【需要的輸入】
- 產業名稱：___________
- 分析維度：___________

【輸出格式】
- 執行摘要（200 字以內）
- 各維度分析（每個維度 300 字以內）
- 結論和建議

【風格】
- 數字導向，每個觀點要有數據支撐
- 決策者視角，幫老闆快速了解重點

【禁止】
- 不要猜測數字，所有數字要標注來源
- 不要過度複雜的圖表
---

檔案：format-guide.md

---
【圖表格式】
- 只用表格和柱狀圖
- 每個圖表不超過 5 行/柱
- 圖表要有標題和數據來源
---
```

## 第三層：專案 Context（專案層）

每個重要專案建立一個 Context 檔，記錄目標、約束、已知決策。

### 專案 Context 的實際案例

```
【專案 Context：東南亞市場拓展】

檔案：southeast-asia-context.md

---
【專案背景】
- 目標市場：越南、泰國
- 預計上線時間：Q3 2026
- 預算：NT$500,000

【已知決策】
- 優先進入越南市場
- 初期不建立當地團隊，用代運營方式
- 主要客群：25-35 歲在職女性

【約束條件】
- 法規：需要當地合作夥伴協助注册
- 文化：需要當地化行銷內容
- 金流：需要串接當地支付方式

【已完成的分析】
- 市場規模：越南約 15 億美元
- 主要競爭者：[名單]

【待確認】
- 當地合作夥伴
- 法規可行性
---
```

## AI 也需要「入職培訓」

> **AI 就像一個新進員工。**

當新人進公司時，你不會丢一堆文件說「自己看」，而是分階段引導。

對 AI 也完全一樣。

### 三階段引導

```
新人進公司               AI 開始新任務
──────────────────────────────────────
1. 公司介紹          →  1. 提供 Context
2. 環境熟悉          →  2. 示範一次流程
3. 工作中指導         →  3. 階梯式 Prompt
```

**第一階段：給 Context**
就像新人要了解公司背景，AI 也需要知道你是誰、要什麼、禁止什麼。

**第二階段：示範一次**
不要只給指令，要示範一次完整的流程。

**第三階段：階梯式 Prompt**
從簡單任務開始，逐步增加難度。

### 入職培訓的實際案例

```
任務：讓 AI 幫你做每週的競品分析

❌ 直接給任務：
「幫我做每週競品分析」

✅ 入職培訓方式：

Step 1：給 Context（公司介紹）
「我要讓你幫我做每週競品分析。
 我們是母嬟用品電商，主要競爭對手是 XXX 和 YYY。
 每週一上午 10 點需要交付。」

Step 2：示範一次（環境熟悉）
「上次我做競品分析的流程是這樣的：
 1. 收集三個競爭對手的社群發文
 2. 比較當週的行銷主題和促銷力度
 3. 產出一頁摘要報告

 請按這個流程分析上週的競品動態。」

Step 3：階梯式 Prompt（工作中指導）
「很好。從下週開始，每週一早上 9 點，
 請按照這個流程自動執行。
 如果有任何異常（例如某對手有大動作），
 特別標注出來。」
```

## 讓 AI 幫你建立長期記憶

當 AI 完成一個任務，表現令你滿意時，你可以問它：

```
「幫我總結這次任務的關鍵 Context，未來類似任務可以直接使用」
```

AI 會生成一個「任務 Context 模板」，你可以存入任務模板庫，未來直接拿出來用。

---

### 這個模式等於「AI 幫你寫 SOP」

```
你：幫我做一個市場分析
AI：[完成得很好]

你：「幫我總結這次分析的 Context，未來類似任務可以直接用」

AI：
```
【任務 Context 模板：市場分析】
- 需要的輸入：產業名稱、競爭者清單
- 輸出格式：SWOT + 三個策略建議
- 風格：決策者視角，數字說話
- 禁止：猜測數字，必須標注數據來源
```
```

你把這個模板存入任務模板庫，下次做市場分析時拿出來直接填空。

---

### 讓 AI 幫你建立 Prompt 庫的實戰對話

```
你：我需要讓你幫我建立一個「商業郵件 Prompt 庫」

AI：好的。請告訴我三個你經常需要寫的商業郵件類型。

你：
1. 跟外部窗口溝通專案進度
2. 向老闆匯報工作成果
3. 跟團隊成員協調任務

AI：明白了。請針對第一個場景，給我一個具體的例子，
 包括：你原本寫的郵件內容、當時的情境、以及你希望達到的目的。

[你提供例子]

AI：很好。根據這個例子，你的「外部窗口溝通郵件」
 Prompt 模板是：

【情境】我是[職位]，要寫一封郵件給[對象]
【目的】溝通[專案名稱]的進度
【現況】目前[目前進度]
【問題】遇到了[什麼問題/延遲]
【需求】需要[對方做什麼]
【語氣】專業、明確、不模糊
【禁止】不要[具體禁止事項]

請用這個模板，針對[你的具體郵件需求]套用。
```

## 分層管理的實際案例：每週例行工作

### 週一：競品分析

```
使用 Context：
□ 系統層：我的身份和偏好
□ 任務層：競品分析 Prompt 模板
□ 專案層：這週的重點關注對手

→ 不需要每次重新輸入背景
→ AI 直接開始執行任務
```

### 週三：專案進度更新

```
使用 Context：
□ 系統層：我的身份和偏好
□ 任務層：進度更新 Prompt 模板
□ 專案層：東南亞市場拓展專案 Context

→ AI 知道我每週三要更新這個專案
→ 自動按照上次建立的流程執行
```

### 週五：數據回顧

```
使用 Context：
□ 系統層：老闆關心的指標
□ 任務層：數據回顧 Prompt 模板
□ 專案層：相關專案 Context

→ AI 知道我關心什麼數據
→ 直接產出我需要的老闆視角回顧
```

## 常見錯誤：把所有東西都塞進上下文

### 錯誤一：建立超長的 AI 身份檔

```
❌ 錯誤：
把 50 頁的公司介紹、十年產品歷史、
所有過往專案全部貼給 AI

→ AI 上下文爆炸
→ 輸出品質反而下降
→ 遺忘真正重要的 Context

✅ 正確：
每次對話只提供「這次任務需要」的 Context
其餘的，存在任務模板庫，需要時呼叫
```

### 錯誤二：建立 Context 但從來不更新

```
❌ 錯誤：
建立了一個「AI 身份卡」，但從來不更新
兩年前的資料還在用

→ AI 拿著過時的 Context 工作
→ 輸出當然不準確

✅ 正確：
每季回顧一次 Context 檔
更新：產業動態、個人目標、優先順序
```

## 本章重點

1. **AI 記憶有代價**——Token 消耗和速度都是成本
2. **分層管理**——系統層 / 任務層 / 專案層
3. **AI 需要「入職培訓」**——Context → 示範 → 階梯式引導
4. **三層資料夾系統**——對話上下文 / 任務模板庫 / 專案 Context
5. **讓 AI 生成長期記憶**——每個好結果，問「這個 Context 能當模板嗎？」
6. **常見錯誤**：Context 太長不更新、過時的資料仍在用
