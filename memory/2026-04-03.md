## 下午工作記錄 (17:00-18:51)

### Xiaomi OpenClaw 配置修復

#### 問題
- Xiaomi `mimo-v2-pro` 模型在 OpenClaw 一直回 401

#### 根本原因
- Kevin 的新 Xiaomi API key (`tp-s99yla3kmqvf30r8ja17ajepq6nzfjb45d2sw0r8cmzyaq5l`) 是 **Token Plan** 類型
- 必須用 endpoint: `https://token-plan-sgp.xiaomimimo.com/v1`
- 不能用預設的 `https://api.xiaomimimo.com/v1`（會 401）

#### API 測試結果
- `POST https://token-plan-sgp.xiaomimimo.com/v1/chat/completions`
  - `mimo-v2-pro` → ✅ HTTP 200
  - `mimo-v2-flash` → ❌ HTTP 403 "Illegal access"
  - Anthropic `/v1/messages` → ❌ 404 Not Found
- **結論**: key 只能用 `mimo-v2-pro`，不支援 flash

#### 修改的檔案
1. `~/.openclaw/.env`
   - `XIAOMI_API_KEY=tp-s99yla3kmqvf30r8ja17ajepq6nzfjb45d2sw0r8cmzyaq5l`
   - `XIAOMI_BASE_URL=https://token-plan-sgp.xiaomimimo.com/v1`
2. `~/.openclaw/openclaw.json`
   - `models.providers.xiaomi.baseUrl` → `https://token-plan-sgp.xiaomimimo.com/v1`
3. `~/.openclaw/agents/main/auth-profiles.json`
   - 新增 `xiaomi:default` profile（之前缺這個）
   - `lastGood.xiaomi = xiaomi:default`
4. `~/.openclaw/openclaw.json`
   - primaryModel: `xiaomi/mimo-v2-pro`（之前是 flash，key 沒 flash 權限）

#### ⚠️ 風險：openclaw onboard 會覆蓋 baseUrl
- 教訓：執行 `openclaw onboard --auth-choice xiaomi-api-key` 後
- Xiaomi `baseUrl` 會被打回預設 `api.xiaomimimo.com/v1`
- 需要手動再改回 `token-plan-sgp.xiaomimimo.com/v1`

#### 驗收狀態 (18:51)
- Gateway 重啟次數: 3 次
- API 直接測試: ✅ HTTP 200
- Log 裡的最後一次 Xiaomi run: 17:39 (401)
- 18:10 重啟後: Xiaomi 尚未成功跑過（一直在 fallback）
- Kevin 正在 Discord 測試是否能正常對話

#### 待驗收
- [ ] Kevin 在 Discord 切到 xiaomi/mimo-v2-pro 能否正常回覆
- [ ] 確認 runtime 不再 401/403/timeout

#### 未來修復指南
如果升級後 Xiaomi 又壞了，檢查順序：
1. `openclaw.json` 的 `models.providers.xiaomi.baseUrl` 是否仍是 `https://token-plan-sgp.xiaomimimo.com/v1`
